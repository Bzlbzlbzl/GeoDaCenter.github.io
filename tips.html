Getting Started: Input, Mapping and Table Operations

What characteristics do my data have to have?

To analyze your data in GeoDa, they should be (information about data size and islands):

  • In shape file (.shp) format (shape files are native to ESRI's ArcView/ArcGIS software. If data are formatted for MapInfo or other programs, they have to be converted or they can be imported into GeoDa from ASCII or dbf format for points and polygons);
  • Continuously distributed (not categorically)
  • Refer to discrete areal units (as opposed to sample or event points)
  • Have no missing values.
  • Cross-sectional data (as opposed to panel data)
  • Contain a unique ID (if no unique ID exists, it can easily be added in a spreadsheet or GIS program; more information)

What options are there to change colors?

Mostly through right-clicking, users can change the color of:

  • a map's default color (in regular and movie format)
  • the legend categories (click on the legend color you want to change)
  • the shading color of the cross-hatched selection, and
  • the background color (e.g., useful for box plots and Moran scatter plots when linking & brushing).

Taking a screenshot of your GeoDa output

GeoDa allows you to export your map or object and legend as separate images: Right-click on the map, go to Save Image as and save it as a .bmp file. Right-click on the legend and choose Copy legend to clipboard. In another program, you can add these two images together, e.g., by inserting them into a 2-column table.

This process is a bit tedious and only includes one GeoDa window. Another way to capture your GeoDa output is to take a screenshot. A simple way to do so is to select Alt-PrtScn on your keyboard and paste the screenshort into another program. For fancier screenshot options, several free and commercial screenshot programs are available (for details, see the May 2004 Openspace archives on exporting maps and legends).

GeoDa currently does not contain the option to save all of your visual output as a project and return to it later. Taking a screenshot can serve as an informal archive of your visual output.

Can I load more than one layer in GeoDa?

Since GeoDa is not designed as a substitute for a GIS, it does not incorporate the option to work with multiple layers. However, it is possible to add a layer to compare shape files after they have been transformed (e.g., from polygons to points or vice versa). However, only the top layer is active. For instance, if the central points of a polygon file are added as a top layer and a quantile map is created, the quantile map colors are applied to the points while the base layer maintains its default color.

What to do with missing values?

GeoDa cannot deal with missing values - it will fill blank fields with zeros or treat values such as 99, -1, etc. as observed. There is no easy solution to this problem. Some options include excluding missing observations, resaving your shape file for only those areas without missing values or interpolating missing values (care needs to be taken that this interpolation is not based on the values of immediate neighbors, otherwise spatial autocorrelation is introduced by design). More information on Openspace.

I have space-time data - what to do?

GeoDa is designed for cross-sectional data. There are a few limited applications of time data in GeoDa. For instance, you can use the bivariate global Moran scatterplot and local Moran (LISA) maps to associate the neighboring values at one time period to the core location of another time period.Other free exploratory spatial data analysis tools exist to analyze space-time data, e.g., Space-Time Analysis of Regional Systems (STARS).

For more information on space-time data in a modelling context, see Openspace (e.g., the spatial panel thread from March 2004 and the August 2004 threads on "error correction models with spatial dependence" and "spatial poisson").

Exploratory (Spatial) Data Analysis

Are there dataset size limits for doing ESDA?

The primary limit to the number of observations in ESDA is visual and hardware-dependent: For instance, unless you are using a very large screen to project your results, you will not be able to visually discern patterns in one million points. You can run regressions and do ESDA in GeoDa with large datasets (several hundred thousand; depending on your hardware and the density of your weights matrix) - it is computationally possible but you often run out of screen real estate when it comes to interpreting the results visually.

Another problem is that you currently cannot change the color of your polygon outlines - as a result, if you have small polygons close together, all you see on smaller screens is the black outline rather than the map-related color of the geographic unit.

Applications for scatter plot matrices

Scatter plot matrices are useful for bivariate Moran scatter plots, e.g., where the neighboring values of one time are regressed on the observations of another time. Alternatively, a matrix can be created with different scatter plot types, e.g., comparing y and x in a regular scatter plot to x-lag and x in a Moran scatter plot, to y-lag and x in a bivariate scatter plot, and so on (for details, see the GeoDa workbook, pp. 61-65 and pp. 160-162).

Assessing the leverage of outliers

Brushing the scatter plot with the Exclude Selected option activated allows you to explore the leverage of outliers by excluding extreme observations and recomputing the slope.

Condition a variable on East-West and North-South

In a conditional map, if you choose x and y coordinates as your x and y variables, you can explore patterns in a variable by changing East-West and North-South thresholds.

Create a map with a continuous color ramp

In a conditional map, if you move all the map dividers and dot levers out to the top and right, you can create a single map with a continuous color ramp (as opposed to quantiles or other intervals).

Why are there differences between my excess risk and raw rate map?

When the distribution of your data is highly skewed, a quantile map based on raw rates can show areas as high that are shown as low values in an excess risk map. This occurs when your median is below the mean since the raw rate map depicts values above the median as high while they show up as below the mean in the excess risk map.

Weights and Spatial Autocorrelation

Which weights matrix should I choose?

The choice of weights should ultimately be driven by a rationale for including those areas as neighbors that have a spatial effect on a given location. This rationale can be derived from theory or be the result of using ESDA to experiment with different weights and connectivity orders. Since weights matrices are used to create spatial lags that average neighboring values, the choice of a weights matrix will determine which neighboring values will be averaged. For instance, since rook weights will usually have fewer neighbors than queen weights, on average, each neighboring observation has more influence.

The question of which weights to choose is more pertinent in the context of modeling than ESDA since modeling is based on substantive notions of spatial effects while ESDA prioritizes the rejection of spatial randomness. Therefore, if there are no substantive reasons to guide the choice of weights in ESDA, using a weights file with as few neighbors as possible (such as rook) makes sense. Especially with irregular areal units (as opposed to grids), the difference between rook and queen weights is often minimal. However, it is advisable to test how sensitive your results are to your weights specifications by comparing multiple weights matrices.

Create your own weights

You can create your own weights file outside of GeoDa but it has to be in binary format. This means that neighbors are defined by zeros and ones, not by other values, e.g., continuous values that reflect a weighting scheme. The weights file has to match the contiguity format used in GeoDa's .gal and .gwt files (see the GeoDa Manuals for details). Note that the third column in the .gwt distance weights file is not currently used for inverse distance weighting since GeoDa uses distance bands.

You can create weights based on criteria other than geographic proximity. However, note that weights are assumed to be exogenous in spatial regression modeling. To avoid endogeneity and circular reasoning, care should be taken to not build too many assumptions and parameters into the weights.

What happens when I set the threshold distance lower than the default?

The default distance threshold ensures that every observation has at least one neighbor. If you set the threshold distance as smaller than the default minimum, GeoDa will create a weights matrix based on your threshold, not the default, i.e., it will contain "island" observations without neighbors. You can easily check this by looking at the weights characteristics in GeoDa and by opening the weights matrix in a text editor.

If you are curious about the difference between the matrix with the default minimum threshold and yours, create both matrices and compare them by looking at their weights characteristics (which is a histogram of the number of neighbors for all observations). You will see that the minimum threshold matrix has no observations with zero neighbors while yours will have observations with zero neighbors.

Does GeoDa include inverse distance weighting?

  • Distance bands
  • K-nearest neighbors

You can create two types of distance-based weights (with .gwt file extensions) in GeoDa:

GeoDa currently does not yet implement inverse distance weighting (i.e., the third column with distances in the .gwt file is only used to determine whether a point is within a distance band or not). The extent of the distance band is specified by your threshold distance (the default value ensures that you have no islands). GeoDa row-standardizes weights, i.e., the rows sum to 1.

If the size of your areas varies considerably, you might be interested in using K-nearest neighbors, which ensures that every point has the same number of neighbors. Note, however, that you cannot yet run or get correct results for spatial lag or error models with KNN weights in GeoDa since these weights are asymmetric.

How do I get the distance units I want?

If you need the metric displayed in the distance weights threshold box to reflect a particular unit (e.g., feet or miles), project your shape file first and set the desired units in another program such as a GIS. Then create the weights based on this file.

What to do with islands?

Islands should not be used for LISA maps and can be problematic for GeoDa's regression models. One option to ensure no islands is to use distance weights (distance bands or k-nearest neighbors). You can also remove islands using a GIS or assign them to other areas that are similar by editing the weights matrix in a text editor (for details on the weights formats, see the GeoDa 0.9.3 User's Guide, pp. 80-81). You will need to assign a mainland area ID to the island ID, and vice versa, assign the island ID to the mainland ID.

How do I project my shape file?

GeoDa does not contain any map projection tools. Map projections can be relevant when using GeoDa's distance weights, which are based on the distances between points. You can set the distance units displayed in the weights distance dialog (e.g., feet, meters, or miles) by first projecting the shape file outside of GeoDa. More information on the distance units.

Project your shape file outside of GeoDa using a GIS or other program. More information on Openspace1 (September 2005) and Openspace2 (September 2005).

LISA maps only show cluster center

Note that the cluster map only shows the center of the cluster in color (e.g., red for a high-high cluster). The actual extend of the cluster includes the center and the surrounding neighbors as defined by the weights matrix. If you are not sure which units are neighbors, open the weights file in a text editor to determine the neighborhood structure.

LISA maps incompatible with islands

Do not create local Moran maps for shape files with islands since the significance levels for the islands will not be computed correctly. You can use distance weights (distance bands or k-nearest neighbors) or exclude the islands from the shape file in a mapping program outside of GeoDa (or edit the shape file manually in ASCII after exporting it from GeoDa). Or you can manually edit your weights file in a text editor and associate the island with the mainland (more information).

Assess sensitivity of LISA results

To assess the sensitivity of your results, run several permutations and apply several significance filters. This way you can explore which clusters and spatial outliers remain stable throughout.

My box map does not reflect my LISA map - why?

The box map's quantile categories, like the box plot's, are based on the median. With highly skewed distributions (e.g., when the mean is much lower than the median), the box map's categories might not reflect the same pattern as the LISA map. When the distribution is symmetric, the box map and LISA map results are very similar.

How robust is the pseudo p-value?

Explore how robust your observed Moran's I value is compared to the randomized reference distribution. In the Moran scatter plot, increase the number of permutations from 99 up and test if the relationship between the observed and randomized Moran's I changes. In the LISA maps, increase the number of randomizations from 99 permutations up to see if the clusters change.

Testing for boundary effects

  • You can explore if the assumption of stationarity is violated by boundary effects. Open the Moran scatter plot (with the Exclude Selected option turned on). Open a map and select the areas next to the boundary of your shape file. Test if the slope/Moran's I changes when the boundary areas are excluded.

Rates and Smoothing

Sensitivity analysis of different smoothers

To assess how sensitive the identification of outliers is to a specific smoothing technique, you can visually compare the different smoothers in GeoDa. Create four maps (raw rate, EB-smoothed, spatial rate, and EBS-smoothed) and choose the box map option under map themes when you choose your base and event variables.

One of the considerations in choosing between smoothers is that the area where strength is borrowed from to correct for local variance instability should be representative of the underlying risk in a local area.

You can also open one or more box plot(s) of the raw rate and/or the smoothed rates to explore the differences in outliers through linking and brushing.

Regression

How can I see the p-value for Moran's I in my output?

To get the p-value for Moran's I in the regression output, check the 3rd box at the initial input window "Moran's I z-value". Once it's checked the N/A changes to a value.

Run regression without opening project

You can access GeoDa's regression functionality without opening a shape file by going directly to Regress after opening GeoDa. This option is particularly useful if you are working with large datasets (e.g., several hundred thousand observations), to avoid long loading times of the map file. Note that this option does not work if the constant is not selected.

Where can I get help interpreting the regression output?

The GeoDa workbook contains several regression chapters with more detail on interpretation of output. For further background reading, see the regression references and, in particular, Anselin (1988).
